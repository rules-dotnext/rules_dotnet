# Phase 3: NuGet Resolution

## Anti-Patterns (this phase)
- NEVER use NuGet `contentHash` as Bazel integrity hash (#7)
- NEVER assume host .NET SDK exists (#11)

## Goal

Make all NuGet package dependencies available as Bazel targets. Choose the resolution strategy that best fits the existing project.

## Decision Tree

```
Has paket.dependencies?
├── YES → Paket Path (Step A)
└── NO
    ├── Has packages.lock.json?
    │   ├── YES → Lock File Path (Step B)
    │   └── NO
    │       └── Direct Package Path (Step C)
```

---

## Step A: Paket Path

Paket is a dependency manager for .NET that replaces NuGet's built-in resolution. If the repo already uses Paket:

### A1. Install paket2bazel

```bash
dotnet tool install paket2bazel  # or use the dotnet_tool rule
```

### A2. Generate Bazel extensions

```bash
paket2bazel --dependencies-file paket.dependencies \
            --output-folder dotnet \
            --output-extension-name "paket.main"
```

This produces `.bzl` extension files that declare all packages.

### A3. Wire into MODULE.bazel

```starlark
paket_extension = use_extension("//dotnet:paket.main_extension.bzl", "paket_main_extension")
use_repo(paket_extension, "paket.main")
```

### A4. Reference packages in BUILD files

```starlark
deps = ["@paket.main//newtonsoft.json"]
```

Package names are lowercase in the Paket hub repo.

## Step B: Lock File Path

If the repo has `packages.lock.json` (generated by `dotnet restore --use-lock-file`):

### B1. Augment with SHA-512 hashes

The lock file contains `contentHash` (SHA-512 of package content) but Bazel needs `nupkgSha512` (SHA-512 of the .nupkg file itself). Use `nuget2bazel` to augment:

```bash
# nuget2bazel adds nupkgSha512 fields to the lock file
dotnet tool run nuget2bazel -- augment packages.lock.json
```

If `nuget2bazel` is not available, compute SRI hashes manually:
```bash
# For each .nupkg in the global cache:
openssl dgst -sha512 -binary package.nupkg | openssl base64 -A | sed 's/^/sha512-/'
```

### B2. Wire into MODULE.bazel

```starlark
nuget = use_extension("@rules_dotnet//dotnet:extensions.bzl", "nuget")
nuget.from_lock(
    name = "nuget",
    lock_file = "//:packages.lock.json",
)
use_repo(nuget, "nuget")
```

### B3. Reference packages in BUILD files

```starlark
deps = ["@nuget//newtonsoft.json"]
```

## Step C: Direct Package Path

When no lock file or Paket exists, declare packages individually:

### C1. Collect all PackageReferences

From Phase 1 reconnaissance, gather every unique `{id, version}` pair across all projects.

### C2. Compute SHA-512 SRI hashes

For each package, download the .nupkg and compute:
```bash
curl -sL "https://api.nuget.org/v3-flatcontainer/{id}/{version}/{id}.{version}.nupkg" \
  | openssl dgst -sha512 -binary | openssl base64 -A | sed 's/^/sha512-/'
```

### C3. Generate nuget.package tags

```starlark
nuget = use_extension("@rules_dotnet//dotnet:extensions.bzl", "nuget")

nuget.package(
    name = "nuget",
    id = "Newtonsoft.Json",
    version = "13.0.3",
    sha512 = "sha512-...",
    tfms = ["net8.0", "netstandard2.0"],
    deps = {
        "net8.0": [],
        "netstandard2.0": [],
    },
)

# Repeat for each package...

use_repo(nuget, "nuget")
```

### C4. Resolve transitive dependencies

Each package's `deps` field must list its direct NuGet dependencies per TFM. Extract these from the `.nuspec` file inside each `.nupkg`:

```xml
<dependencies>
  <group targetFramework="net8.0">
    <dependency id="SomeOtherPackage" version="1.2.3" />
  </group>
</dependencies>
```

## Common Issues

### "integrity mismatch"
The SHA-512 hash doesn't match. Re-download the .nupkg and recompute. NuGet packages are immutable — if the hash changed, the download was corrupted.

### Missing transitive dependencies
If `bazel build` fails with missing package errors, a transitive dependency was not declared. Add it to the NuGet hub.

### Version conflicts
If two packages require different versions of the same transitive dependency, declare the higher version. rules_dotnet's depset ordering ensures the most direct dependency wins.

### Private NuGet feeds
Use the `sources` attribute for custom feeds and `netrc` for authentication:
```starlark
nuget.from_lock(
    name = "nuget",
    lock_file = "//:packages.lock.json",
    sources = [
        "https://api.nuget.org/v3/index.json",
        "https://pkgs.dev.azure.com/myorg/_packaging/myfeed/nuget/v3/index.json",
    ],
    netrc = "//:.netrc",
)
```

## Verification Gate

```bash
# For Paket path:
bazel query @paket.main//...

# For lock file / direct path:
bazel query @nuget//...
```

This should list all expected packages as available targets. Compare against the PackageReference list from Phase 1.
